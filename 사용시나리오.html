방과후 스마트 정산 시스템 · 온보딩 시나리오

처음 쓰는 담당자가 1분기를 끝까지 마감하기까지

사용설명서가 화면별 기능을 사전처럼 설명한다면, 이 문서는 실제로 손을 움직이는 순서입니다. 한 학교의 담당 선생님이 시스템을 처음 켜서 1분기 정산을 마감하고 청구서를 뽑기까지, 9단계로 나눠 선생님이 하는 일과 시스템이 대신 하는 일을 구분해 정리했습니다.

학교
늘봄초등학교 · 방과후 담당 선생님 (처음 사용)
회계 유형
교재통합형 (수강료 + 교재비 2단)
예시 강좌
로봇과학 — 2개 반(A·B), 월 강사료 100,000원, 기초 교재비 30,000원
예시 학생
김하늘 (3학년 2반, 초3 대상) · 이서준 (2학년 1반, 자유수강권 대상)
선생님이 하는 일 시스템이 자동으로 하는 일

전체 흐름 (클릭하면 해당 단계로 이동)

0시작 전 준비 → 1회계 유형 선택 → 2부서·강좌 등록 → 3수강생 명단 등록 지원금 대상 확정 → 4자유수강권·초3 확인 → 5정산 결과 확인 → 6예외 처리(조정·환불) 분기 마무리 → 7차수 마감 → 8서식 생성·백업 ⟲ 다음 분기 시작 — 강좌·요금이 바뀌었나요? ↺바뀐 게 있으면 → 1스텝부터 / ↺그대로면 → 2스텝부터
0
실행 전

시작 전 준비

아직 시스템 화면을 보기 전 단계. 어떤 방식으로 열지만 정하면 됩니다.

🙋 선생님이 하는 일
  • 방과후 스마트 정산 시스템 https://naetc.github.io/smartafterschool/ 에 접속
  • 가급적 크롬 또는 엣지로 열기 — 엑셀 생성·이미지 저장 기능이 가장 안정적으로 동작
⚙ 시스템이 하는 일
  • 브라우저 저장소(IndexedDB)에 저장된 데이터가 있는지 확인
  • 저장된 데이터가 없으면 → 웰컴 화면(1단계) 자동 표시
  • 저장된 데이터가 있으면 → 웰컴 화면을 건너뛰고 마지막으로 보던 화면으로 진입
Tip이 판단은 브라우저 단위입니다. 다른 브라우저로 열면 처음 화면부터 다시 시작됩니다.
1
웰컴 화면

회계 유형 선택

시스템 전체의 데이터 구조를 정하는, 되돌릴 수 없는 단 한 번의 선택.

🙋 선생님이 하는 일
  • 재료비를 따로 청구하는 학교인지 확인 → 교재통합형 또는 교재분리형 카드 선택
  • 이 예시는 교재통합형으로 진행 → "교재통합형으로 실무 장부 만들기" 클릭
  • (연습이 먼저 필요하면 "튜토리얼 체험하기"로 가상 데이터부터 눌러볼 수 있음)
⚙ 시스템이 하는 일
  • 회계 유형(accType)과 공제 우선순위 기본값을 확정해 저장
  • 1~4분기 빈 장부를 생성하고 1스텝 · 1분기 화면으로 이동
  • 이후 이 선택은 화면 안에서 다시 바꿀 수 없도록 잠금
주의바꾸려면 환경설정 → 시스템 초기화로 지금까지 쌓인 기록을 지우고 처음부터 다시 시작해야 합니다.
2
1스텝 · 부서/강좌 세팅

부서·강좌 등록

청구의 기준 단가를 만드는 단계. 여기서 정한 값이 이후 모든 계산의 뿌리가 됩니다.

🙋 선생님이 하는 일
  • 부서명 로봇과학, 강좌수 2(반 수), 월 강사료 100,000, 월 수용비 0, 기초 교재비 30,000 입력
  • 주간단위 1, 차수별시수 4,4,4 입력 (팝오버로 차수별 스피너 조정 가능)
  • "개별 추가" 클릭 — 강좌가 많으면 엑셀 양식을 받아 한 번에 업로드
⚙ 시스템이 하는 일
  • 강좌수만큼 로봇과학(A), 로봇과학(B) 개별 강좌를 자동 복제
  • 월 강사료·수용비를 반 수만큼 균등 배분해 강좌 요금표에 채움
  • 주간단위·차수별시수로 이번 분기 실제 청구액을 비례 계산 (105,000원급)
  • 같은 값을 1~4분기 전체에 동일하게 반영
확인강좌 요금표에서 로봇과학(A), 로봇과학(B) 두 줄이 생겼는지, 총 수강료가 계산돼 있는지 확인.
3
2스텝 · 수강생 명단 관리

수강생 명단 등록

학생을 강좌에 실제로 연결하는 단계. 이 순간부터 그 학생의 청구액이 계산되기 시작합니다.

🙋 선생님이 하는 일
  • 개별 등록: 학년 3 · 반 2 · 번호 5 · 이름 김하늘, 강좌 로봇과학(A) 선택 후 "수강생 추가"
  • 인원이 많으면 통합/강좌별 명단 엑셀을 받아 업로드 (기존 명단에 추가 / 덮어쓰기 중 선택)
  • 1스텝에서 폐강 처리로 생긴 미배정(누락) 학생이 있으면 여기서 다른 강좌로 재배정
⚙ 시스템이 하는 일
  • 학생을 로봇과학(A) 강좌에 연결하고 실부담금(지원 전) 계산을 즉시 실행
  • 분기별/강좌별 현황판의 인원수를 실시간으로 갱신
  • 강좌를 못 찾은 행은 자동으로 미배정(누락) 명단으로 분리
확인현황판에서 로봇과학(A) 인원이 1명 늘었는지, 미배정 명단이 0건인지 확인.
4
3스텝 · 자유수강권 관리

자유수강권·초3 대상 확인

지원금이 실제로 얼마나 깎아줄지를 결정하는, 이 학생만의 "지원 자격" 단계.

🙋 선생님이 하는 일
  • 자유수강권 탭에서 이서준(2학년 1반 9번)을 개별 등록 — 학년/반/번호/이름만 입력
  • 초3 지원금 탭은 조회만 — 2스텝에서 3학년으로 등록된 김하늘이 이미 자동으로 대상에 잡혀 있는지 확인만
  • 학기 중 전입생이 있다면 "전입생 통합 콘솔"에서 실제 남은 한도를 수동 입력
⚙ 시스템이 하는 일
  • 이서준에게 연간 자유수강권 한도 600,000원을 배정
  • 김하늘은 학년 정보만으로 초3 대상 여부를 자동 판별 (별도 등록 불필요)
  • 전입 한도를 입력하면 그 값 기준으로 이후 정산을 즉시 재연산
확인상단 "전입생 조정 현황" 배지가 방금 조정한 인원수와 일치하는지 확인.
5
4스텝 · 정산 마감 및 예외처리

정산 결과 확인

지금까지 입력한 모든 값이 실제 금액으로 계산돼 나오는, 이 시스템의 핵심 화면.

🙋 선생님이 하는 일
  • 학생별 탭에서 김하늘·이서준 행의 최종징수(자부담) 금액이 상식적인지 눈으로 확인
  • 이상해 보이면 필터를 전체 → 초3지원 또는 자유수강으로 좁혀서 다시 확인
  • 강좌별 · 분기별 탭에서 로봇과학 전체 집계도 함께 확인
⚙ 시스템이 하는 일
  • 초3지원금을 먼저 다 쓰고, 그다음 자유수강권을 쓰는 고정 순서로 자동 공제
  • 같은 지원금 안에서는 환경설정에 정한 순서(예: 수강료 → 교재비)로 소진
  • 김하늘: 105,000원 → 초3 공제 → 0원 자부담
  • 이서준: 105,000원 → 자유수강권 공제 → 0원 자부담
Tip숫자가 이상하면 상단 분기 탭이 원하는 분기인지부터 확인하세요 — 가장 흔한 착시 원인입니다.
6
학생 통합 콘솔

예외 처리 — 조정 · 환불

규칙대로는 안 맞아떨어지는 실제 사례(결석, 자퇴, 할인)를 처리하는 단계.

🙋 선생님이 하는 일
  • 학생별 탭에서 김하늘 이름 클릭 → 학생 통합 콘솔 열기
  • "환불 및 결석 처리"에서 사유 결석(일할계산), 2차수, 결석 시수 4 입력
  • 예상 환불액을 확인하고 "환불 승인" 클릭
  • (다자녀 할인 등 별도 사유가 있으면 "실부담금 강제 조정"에서 금액과 사유를 직접 입력)
⚙ 시스템이 하는 일
  • 결석 시수만큼 시간당 단가로 환불액을 자동 계산해 미리보기로 표시
  • 승인 즉시 김하늘의 정산 내역과 산출근거에 "환불:2차수 결석(4시수)" 반영
  • 학생별·강좌별·분기별 탭의 합계 금액을 전부 재연산
확인산출근거 칸에 방금 처리한 사유가 영문 코드가 아니라 한글 문구로 정확히 남았는지 확인.
7
4스텝 · 시스템마감

차수 마감

이번 차수의 숫자를 확정하고, 실수로 못 건드리게 잠그는 단계.

🙋 선생님이 하는 일
  • 5~6단계에서 확인·수정을 마친 뒤, 상단 시스템마감에서 1차 체크
  • 다음 차수로 넘어갈 때도 반드시 1차 → 2차 → 3차 순서로만 진행
⚙ 시스템이 하는 일
  • 마감 시점의 금액을 스냅샷으로 저장
  • 해당 차수의 모든 입력칸을 잠가 수정 불가 상태로 전환
주의해제할 때는 반대로 가장 최근 차수부터 역순으로만 가능합니다. 중간 차수만 골라 풀 수 없어요.
8
5스텝 · 서식 자동생성

서식 생성 · 백업

여기까지의 결과를 실제로 학교 밖(강사, 학부모, 행정실)에 내보내는 마지막 단계.

🙋 선생님이 하는 일
  • "교육비 청구서(업체)" 탭에서 로봇과학 강사비 정산 확인 후 "엑셀 파일 생성"
  • 학부모용 "수납요구서", "환불 이력서"도 같은 방식으로 생성
  • 화면 상단 백업 버튼을 눌러 오늘 작업분을 JSON 파일로 저장해 둠
⚙ 시스템이 하는 일
  • 지금까지의 정산 데이터를 서식 종류에 맞게 집계 · 가공해 엑셀로 즉시 생성
  • 백업 시점의 전체 데이터(부서·학생·정산 내역)를 하나의 JSON 파일로 내보냄
Tip브라우저 캐시가 지워지면 데이터가 사라질 수 있으니, 마감마다 백업하는 습관을 들이세요.
↺ 2분기가 시작되면? 무조건 3단계부터는 아닙니다 — 강좌 구성이 바뀌었는지가 기준입니다.

① 강좌·요금이 1분기와 똑같다면: 2단계(1스텝)는 건너뛰고 3단계(2스텝)의 "이전 분기 명단 가져오기"로 바로 시작합니다. 같은 강좌가 이번 분기에도 있으면 학생이 자동 배정되고, 강좌가 없어졌으면 미배정(누락)으로 떨어지니 재배정만 해주면 됩니다.

② 강사료가 바뀌었거나, 강좌가 새로 생기거나 없어졌다면: 반드시 2단계(1스텝)부터 먼저 손봐야 합니다. 특히 새 강좌를 추가하면 1~4분기 전체에 한꺼번에 반영되므로, 이번 분기에만 없어야 할 강좌라면 해당 분기의 "운영" 체크박스로 개별적으로 꺼 줘야 합니다.

두 경우 모두 1단계(회계 유형 선택)만은 다시 거칠 필요가 없습니다 — 처음 한 번만 정해지는 값입니다.
